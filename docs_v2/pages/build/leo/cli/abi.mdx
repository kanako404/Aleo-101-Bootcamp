# `leo abi`

从已编译的 `.aleo` 字节码文件生成 ABI（Application Binary Interface，应用程序二进制接口）。

ABI 是一个 JSON 格式的接口描述文件，用于说明程序的公共接口，包括 transitions、mappings 和 types。借助 ABI，下游工具可以在没有原始源代码的情况下，与已经部署的程序进行交互。

与 `leo build` 自动生成的 ABI 不同，`leo abi` 命令会直接基于 `.aleo` 字节码文件生成 ABI。这在以下场景中非常有用：

- 你需要为并非自己编写的程序生成 ABI；
- 你需要检查或分析已经部署到链上的程序；
- 你需要基于已有的 `.aleo` 字节码生成可供前端、脚本或其他工具使用的接口描述文件。

## 使用方式

```bash
leo abi <FILE>
```

## 示例

```bash
leo abi build/main.aleo
```

## 参数说明

### `<FILE>`

需要生成 ABI 的 `.aleo` 字节码文件路径。

该参数是必填项。

## 可选参数

### `--network <NETWORK>`

### `-n <NETWORK>`

指定用于解析的网络。

可选网络包括：

```bash
testnet
mainnet
canary
```

默认值为：

```bash
testnet
```

## 输出参数

### `--output <OUTPUT>`

### `-o <OUTPUT>`

指定 ABI JSON 文件的输出路径。

如果没有指定该参数，生成的 ABI 会直接打印到终端标准输出中。

## 使用示例

### 1. 生成 ABI 并打印到终端

```bash
leo abi build/main.aleo
```

### 2. 为指定网络生成 ABI

```bash
leo abi build/main.aleo --network mainnet
```

### 3. 将 ABI 保存为 JSON 文件

```bash
leo abi build/main.aleo --output abi.json
```

### 4. 从导入的依赖程序中生成 ABI

```bash
leo abi build/imports/credits.aleo -o credits_abi.json
```

## 补充说明

ABI 可以理解为程序对外暴露的“接口说明书”。它不会包含完整的源代码逻辑，但会描述外部工具如何调用程序、读取程序结构，以及识别程序中可交互的公开接口。

在实际开发中，ABI 通常用于以下场景：

- 前端页面调用链上程序；
- SDK 或脚本与已部署程序交互；
- 检查第三方程序的公开接口；
- 为部署后的程序生成标准化接口文件；
- 将 `.aleo` 字节码与外部工具链连接起来。

如果需要进一步了解 ABI 的格式、字段结构和类型系统，可以继续查看 ABI Generation guide。
